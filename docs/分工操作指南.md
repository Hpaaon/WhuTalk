WhuTalk 项目本地复现指南（组员版）
前提条件
已收到组长的 GitHub 仓库邀请并接受

本地已安装 Python 3.8 及以上版本

第一步：克隆项目代码
打开终端（命令行），进入你打算存放项目的目录，执行：

``` bash
git clone https://github.com/Hpaaon/WhuTalk.git
cd WhuTalk
```



第二步：创建并激活虚拟环境
Windows（CMD）：

``` bash
python -m venv venv
venv\Scripts\activate
```

激活成功后，命令行开头会显示 (venv) 标识。

第三步：安装项目依赖
``` bash
pip install -r requirements.txt
```
等待安装完成（约 1-2 分钟）。

第四步：初始化数据库
首次运行时会自动建表。直接在终端执行：

``` bash
python app.py
```

你会看到类似输出：

text
✅ 数据库初始化成功！
 * Running on http://127.0.0.1:5000
这说明数据库（social.db）已自动创建，三张表（users、posts、friendships）已就绪。

此时按 Ctrl + C 暂停服务器，或保持运行继续测试。

第五步：验证功能是否正常
确保虚拟环境已激活，执行：

``` bash
python app.py
打开浏览器访问 http://127.0.0.1:5000
```

测试注册 → 登录 → 退出，验证认证模块是否正常工作。

第六步：开始你的开发工作
6.1 创建自己的开发分支（重要！）
为了避免大家同时修改主分支造成冲突，请务必在开始写代码前创建自己的特性分支：

``` bash
# 先切换到 main 分支并拉取最新代码
git checkout main
git pull origin main

# 创建并切换到你的特性分支（例如负责个人资料模块）
git checkout -b feature/你的模块名
```

常用分支命名参考：

个人资料模块：feature/profile

动态发布模块：feature/post

好友管理模块：feature/friend

时间线聚合模块：feature/timeline

6.2 日常开发流程
每天开始工作前，先拉取组长合并的最新代码：

```bash
git checkout main
git pull origin main
git checkout feature/你的模块名
git merge main   # 将主分支的最新更新合并到你的特性分支
```

6.3 提交代码
当你完成一个功能点（或一天工作结束）时：

```bash
# 查看修改了哪些文件
git status

# 添加所有修改
git add .

# 提交（写清楚本次做了什么）
git commit -m "feat: 完成个人资料编辑功能"

# 推送到远程仓库（你的特性分支）
git push origin feature/你的模块名
```

6.4 发起 Pull Request（PR）
当你的模块开发完成并自测通过后：

打开 GitHub 仓库页面

点击 "Compare & pull request" 按钮

填写 PR 标题和描述（说明你完成了哪些功能）

点击 "Create pull request"

通知组长进行代码审查和合并

常见问题排查

|问题|解决方法|
|---|---|
|python 命令找不到|检查 Python 是否安装，或尝试使用 python3 代替|
|虚拟环境无法激活（Windows PowerShell）|改用 CMD 执行 venv\Scripts\activate，或参考组长发的解决方案|
|安装依赖时报错|确保网络通畅，可尝试使用国内镜像源：pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple|
|social.db 文件不存在|运行 python app.py 会自动创建，无需手动操作|
|端口 5000 被占用|修改 app.py 最后一行，将 port=5000 改为其他端口（如 port=5001）|
|Git 推送失败（权限拒绝）|确保已接受组长的仓库邀请，且 GitHub 账号已配置 SSH Key 或使用 HTTPS 方式认证|
